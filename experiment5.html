Here, we take advantage of naming grid areas, using the grid-area property. Every grid item in the wrapper has a name, which can be addressed in the areas property. This provides an alternative to specifying the grid-area property in terms of numbers or row/column names, as before. The leads to nicely "semantic" specification of the grid structure. A period can be used to signify empty space in the grid. Here, media queries are used to reorganize the content of the grid when the media width is below a certain ceiling. Using names in grid-area and grid-template-areas makes the CSS that handles this implementation of a basic responsive design strategy eminently readable. Here's the relevant part of the CSS (view the source for the full story.). display: flex is only used within the navbar div to arrange the content; everything else falls out of display: grid. This demonstrates how flexbox and CSS Grid can co-exist peacefully. 

/* [...] */

.wrapper {
    display: grid;
    padding: 0.5rem;
    margin: 0.5rem;
    grid-gap: 10px;
    grid-template-columns: 0.3fr 1fr 1fr;
    grid-template-areas: ". header header" "navbar content content" "footer footer footer";
}

.navbar {
    grid-area: navbar;
    display: flex;
    flex-direction: column;
    justify-content: flex-start;
}

/* [...] */

@media (max-width: 960px) {
    .wrapper {
        grid-template-columns: 1fr;
        grid-template-areas: "header" "navbar" "content" "footer";
    }

    .navbar {
        flex-direction: row;
        justify-content: space-around;
    }
}
